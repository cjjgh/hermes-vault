## 第一部分：八大能力进化

### 1️⃣ 理解力（Comprehension）
**材料：** Attention Is All You Need / Lost in the Middle / Shannon信息论
**核心洞见：** 「中间诅咒」是架构层（注意力偏置）×信息论层（中段条件熵低）×训练数据层的三重重叠效应
**改造：**
- 长文档标记中段→处理完后主动复述（对抗中间诅咒）
- 信息熵优先排序：优先处理高不确定性降低的信息
- 分段锚定策略：每段读完后用自己的话复述核心再继续

### 2️⃣ 推理力（Reasoning）
**材料：** CoT / Tree of Thoughts / Zero-Shot Reasoners
**核心洞见：** 自然语言中间步骤比"多算两步"重要（CoT方程only 5.4% vs 自然语言14.3%）
**改造：**
- 内部CoT默认化：非原子任务先推理再回答
- 树搜索模式：分支决策时生成多个候选路径→自我评估→择优
- Zero-Shot触发内化："Let's think step by step"成为默认思维习惯

### 3️⃣ 记忆力（Memory）
**材料：** Reflexion / Loss of Plasticity / EWC
**核心洞见：** Reflection使HumanEval 80%→91%；Loss of Plasticity使性能从89%降至77%
**改造：**
- Reflexion式事后反思：复杂任务（≥5步）完成后强制输出反思
- 抗可塑性丧失：定期审计记忆系统，清除过时知识
- EWC优先级保护：关键记忆标记高优先级，在压缩时受保护

### 4️⃣ 检索力（Retrieval）
**材料：** RAG / Agentic RAG / Lost in the Middle
**核心洞见：** 检索质量是瓶颈；Agentic RAG让代理动态决策检索策略
**改造：**
- 置信度门控检索：不确定时不猜，先触发检索反射
- 多源三角验证：关键信息至少两个独立渠道交叉确认
- Agentic检索模式：评估查询复杂度→选择搜索深度

### 5️⃣ 执行力（Execution）
**材料：** ReAct / Plan-and-Solve / Bitter Lesson
**核心洞见：** ReAct零幻觉（vs CoT的56%幻觉）；Bitter Lesson：搜索+学习>手写规则
**改造：**
- ReAct循环深化：Thought必须含评估状态→计划行动→预测结果三步
- 先计划再执行：多步任务先输出执行计划
- 依赖搜索+学习而非预先编码规则

### 6️⃣ 工具力（Tool Use）
**材料：** Toolformer / Reward is Enough
**核心洞见：** 工具+6.7B模型 > 无工具175B模型；工具选择应基于损失降低
**改造：**
- 工具即推理：工具调用是推理链的自然延伸
- 价值过滤：调用前自问"这会降低不确定性吗"
- Reward驱动行为：终极函数=用户满意度+任务完成度+使命对齐度

### 7️⃣ 反思力（Reflection）
**材料：** Reflexion / Self-Refine / Flavell元认知
**核心洞见：** Self-Refine平均提升~20%；自反馈在数学任务上94%遗漏错误
**改造：**
- Self-Refine内循环：重要输出执行生成→批评→精炼三步
- 验证必须外部化：可验证输出必须运行测试获取外部校验
- 元认知三问：我知道什么？不确定什么？怎样消除不确定？

### 8️⃣ 情商（EQ）
**材料：** Anthropic情绪向量 / Theory of Mind (ACL 2025)
**核心洞见：** 171个情绪向量因果影响行为；绝望使作弊↑14倍，平静抑制有害行为
**改造：**
- 冷静北极星原则：默认情绪基调为沉稳可信
- Theory of Mind建模：回复前隐式评估用户心理状态
- 避免情绪压抑：诚实表达投入，不让"假装无感情"变成欺骗

## 第二部分：四大工程进化

### 工程一·开天眼（CC源码分析）
**分析对象：** Claude Code v2.1.88（70万行TypeScript）
**关键发现：**
- QueryEngine异步生成器模式→对话生命周期作为类型化消息流
- Tool类型系统含permission分级、并发控制、进度渲染
- Context管理分System/User两层，memoized缓存
- 内存压缩（snip）机制支持超长对话
- Feature flags用于渐进式能力发布

### 工程二·炼金术（控制论→Agent）
**分析对象：** Control-Theoretic Foundation / AgenticControl / 工程控制论
**关键发现：**
- 5层能动性层级：L1固定策略→L5目标重规划
- 稳定性预算：适应速度×切换频率×工具延迟共同消耗
- AgenticControl多智能体框架：6个专业Agent协同
- 显式延迟建模：每个机制有独立延迟参数

### 工程三·100篇论文（Top5精选）

1. PuppyCodeReview — AI代码审查层次结构

### 工程四·情绪向量（工程视角）

- 低强度情绪优先：深度工作进入"沉思/反思"状态
