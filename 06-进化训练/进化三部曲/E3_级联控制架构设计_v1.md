# 万手Claw 级联控制架构设计 v1 (E3 §4.2.2)

> 设计时间：2026-05-17 | 基于：工程控制论进化设计方案 E3 §4.2.2

---

## 一、架构概述

将当前扁平控制结构重构为 **三环级联控制架构**，不同时间尺度的控制器各司其职：

```
⏱ 秒级 — 内环：工具调用控制
  ┌────────────────────────────────────────┐
  │ 观测：工具返回结果 + 延迟 τ            │
  │ 控制：失败重试 + 超时处理 + 工具切换   │
  │ 设定点：工具成功率 > 95%               │
  └────────────────────────────────────────┘
        ↑ 测量值           ↓ 控制指令
⏱ 分级 — 中环：记忆与上下文控制
  ┌────────────────────────────────────────┐
  │ 观测：记忆容量 + ρ + σ + τ            │
  │ 控制：写入节流 + 压缩触发 + 归档调度   │
  │ 设定点：ρ×σ×τ < 0.8                   │
  └────────────────────────────────────────┘
        ↑ 测量值           ↓ 控制指令
⏱ 小时级 — 外环：进化与自修复控制
  ┌────────────────────────────────────────┐
  │ 观测：日健康审计 + 能力差距分析        │
  │ 控制：配置更新 + Skill沉淀 + 模型切换  │
  │ 设定点：系统能力趋势向上               │
  └────────────────────────────────────────┘
```

## 二、各环控制律定义

### 2.1 内环 — 工具调用控制（秒级）

| 项目 | 实现 |
|------|------|
| **观测器** | `stability_observer.py`: 记录每次工具调用的 success/latency |
| **控制器** | `tool_guardrails.py`: 连续失败检测 + 重试逻辑 |
| **执行器** | `run_agent.py` 的 `_execute_tool_calls_sequential` |
| **设定点** | 工具成功率 > 95%，单次延迟 < 120s |
| **控制周期** | 每次工具调用（实时） |

**控制律（P 控制）：**
```
e(t) = setpoint_success_rate - measured_success_rate
if consecutive_failures >= 3:
    action = "switch_to_fallback_model"
elif e(t) > 0.05:
    action = "increase_retries"
```

### 2.2 中环 — 记忆与上下文控制（分钟级）

| 项目 | 实现 |
|------|------|
| **观测器** | `rate_controller.py + dwell_time_controller.py`: ρ, σ, τ |
| **控制器** | `event_safety_controller.py`: 耦合积监测 |
| **执行器** | `control_integration.py`: 4个钩子注入 |
| **设定点** | ρ×σ×τ < 0.8 |
| **控制周期** | 每轮 Agent 循环（~分钟级） |

**控制律（PI 控制）：**
```
e(t) = 0.8 - coupling_product(t)
ρ_throttle = Kp * e(t) + Ki * ∫e(t)dt

if coupling_product >= 0.8:
    action = "THROTTLE_WRITES"
elif coupling_product >= 1.0:
    action = "TRIGGER_DEGRADATION"
elif e(t) > 0.2 for 5 cycles:
    action = "RESUME_NORMAL"
```

### 2.3 外环 — 进化与自修复（小时/天级）

| 项目 | 实现 |
|------|------|
| **观测器** | `capability_gap_feedback.py`: 能力差距分析 |
| **控制器** | `agentic_control_optimizer.py`: 参数自优化 |
| **执行器** | Cron 任务（每日健康审计 + 进化执行） |
| **设定点** | 系统能力趋势向上 |
| **控制周期** | 每日一次（cron） |

**控制律：**
```
趋势检测:
  if avg_composite_score(t) > avg_composite_score(t-7d):
      continue current strategy
  else:
      trigger "capability_gap_feedback" → adjust training focus
```

## 三、环间耦合

```
内环测量值 → 中环输入
  tool_results → ρ, σ, τ 计算
  failure_rate → coupling_product
  
中环控制 → 内环设定点更新
  throttling_decision → tool_guardrails 的 max_retries
  compression_trigger → context_limit_ratio

外环控制 → 中环/内环设定点调整
  parameter_optimization → dwell_time_gamma, max_retries
  capability_gap → training_focus
```

## 四、稳定性分析

### 4.1 时间尺度分离

时间尺度至少差 10x 避免环间耦合：

| 环 | 周期 | 带宽 | 默认值 |
|----|------|------|--------|
| 内环 | ~1s | 1 Hz | — |
| 中环 | ~60s | 0.017 Hz | — |
| 外环 | ~86400s | 1.16e-5 Hz | — |

→ 时间尺度分离比 > 60x，满足级联控制设计准则。

### 4.2 饱和保护

每个环的输出应设限幅，防止控制指令溢出导致子系统失稳：

| 控制输出 | 下限 | 上限 |
|---------|------|------|
| `max_retries` | 1 | 10 |
| `compress_threshold` | 0.70 | 0.99 |
| `ρ_max` | 0.10 | 0.50 |
| `dwell_gamma` | 0.05 | 0.50 |
